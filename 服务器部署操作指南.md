# 球星球迷合照 API 服务器部署操作指南

更新时间：2026-04-12  
适用范围：`球星球迷合照/server` 后端服务部署到 Linux 服务器。

## 1. 当前部署结论

项目已经封装为后端 API，核心接口如下：

- 提交任务：`POST /api/v1/synthesis/submit`
- 查询任务：`GET /api/v1/synthesis/query/:taskId`
- 健康检查：`GET /health`

但正式部署前必须处理这些点：

- 服务器必须有 Redis，当前服务启动时会连接 Redis + BullMQ 队列。
- `.env` 中必须配置真实密钥，不能继续使用 `your_server_api_key_here` 这类占位值。
- H5 成功回调字段已按交接文档调整为 `task_id` + `user_image`。
- 提交接口成功响应已按交接文档固定返回 `status: "processing"`。
- 当前结果图直接使用 Seedream/火山侧 URL，自有 OSS/CDN 上传还未实现。

## 2. 服务器要求

建议配置：

- 系统：Ubuntu 22.04 或 Ubuntu 24.04
- CPU：2 核以上
- 内存：4 GB 以上，建议 8 GB
- 磁盘：20 GB 以上
- Node.js：建议 20 或 22 LTS，不建议低于 20
- Redis：6 以上
- 网络：服务器必须能访问 Seedream/火山引擎 API 和 LAS 中转接口

需要开放端口：

- 对外只开放 `80/443`
- Node 服务端口 `3000` 只建议监听本机或内网
- Redis `6379` 不要暴露公网

## 3. 安装系统依赖

登录服务器后执行：

```bash
sudo apt update
sudo apt install -y curl git build-essential nginx redis-server
sudo systemctl enable --now redis-server
redis-cli ping
```

如果返回：

```text
PONG
```

说明 Redis 正常。

安装 Node.js 可以使用公司标准方式，也可以使用 NodeSource：

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

## 4. 上传项目代码

推荐部署目录：

```bash
sudo mkdir -p /opt/bayern-fan-photo
sudo chown -R $USER:$USER /opt/bayern-fan-photo
```

需要上传整个 `球星球迷合照` 项目目录，至少要包含：

- `server/src`
- `server/package.json`
- `server/package-lock.json`
- `素材`
- `参考图`
- `server/src/data`

不要直接上传本机的 `server/node_modules`。服务器上重新安装依赖，避免 Windows 和 Linux 原生依赖不兼容，尤其是 `sharp`。

示例上传方式：

```bash
rsync -av \
  --exclude 'server/node_modules' \
  --exclude 'server/.env' \
  --exclude '测试*' \
  "球星球迷合照/" \
  user@your-server:/opt/bayern-fan-photo/
```

注意：项目里有中文文件名和特殊字符文件名，上传时要确保 UTF-8 文件名没有被 FTP/SFTP 工具改坏。

## 6. 配置环境变量

复制配置文件：

```bash
cd /opt/bayern-fan-photo/server
cp .env.example .env
chmod 600 .env
nano .env
```

推荐生产配置示例：

```bash
# Seedream native 模式
SEEDREAM_MODE=native
SEEDREAM_NATIVE_API_KEY=填真实密钥
SEEDREAM_NATIVE_API_URL=https://ark.cn-beijing.volces.com/api/v3/images/generations
SEEDREAM_NATIVE_MODEL=doubao-seedream-4-5-251128

# 用户照片解读仍会用 LAS / 视觉模型
LAS_API_KEY=填真实密钥
LAS_BASE_URL=填真实地址
VISION_MODEL=gemini-2.5-flash

# 服务配置
PORT=3000
NODE_ENV=production

# 单机部署可先用 true；生产分离 API/Worker 建议 false
ENABLE_EMBEDDED_WORKER=true

# 生产环境必须设置真实值，H5 后端请求时带 x-api-key
SERVER_API_KEY=生成一个强随机字符串

# H5 提供的返回结果接口地址
H5_CALLBACK_URL=https://h5.example.com/api/aigc/callback

# Redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0
# 如果使用云 Redis，可以改用 REDIS_URL
# REDIS_URL=redis://default:password@redis-host:6379/0

# 队列和任务保留
MAX_CONCURRENT=3
TASK_RETENTION_SECONDS=7200
BULLMQ_JOB_ATTEMPTS=1
BULLMQ_JOB_BACKOFF_MS=0
```

重点说明：

- 即使 `SEEDREAM_MODE=native`，当前 `userDescriber.js` 仍然依赖 `LAS_API_KEY` 做用户照片解读，所以 `LAS_API_KEY` 也要配置。
- `SERVER_API_KEY` 不要使用默认占位值，否则代码会当作未开启鉴权。
- 按交接文档，H5 回调地址应配置为 `H5_CALLBACK_URL`。

## 8. 使用 PM2 部署

安装 PM2：

```bash
sudo npm install -g pm2
```

### 方案 A：单进程部署

适合初期联调和低并发部署。

`.env` 中设置：

```bash
ENABLE_EMBEDDED_WORKER=true
```

启动：

```bash
cd /opt/bayern-fan-photo/server
pm2 start src/app.js --name bayern-fan-api --time
pm2 save
pm2 startup systemd
```

执行 `pm2 startup systemd` 后，PM2 会输出一条 `sudo env ... pm2 startup ...` 命令，需要复制执行一次。

### 方案 B：API 和 Worker 分离部署

适合正式生产部署。API 负责接请求，Worker 负责消费队列生成图片。

`.env` 中设置：

```bash
ENABLE_EMBEDDED_WORKER=false
```

启动：

```bash
cd /opt/bayern-fan-photo/server
pm2 start src/app.js --name bayern-fan-api --time
pm2 start src/worker.js --name bayern-fan-worker --time
pm2 save
pm2 startup systemd
```

查看状态：

```bash
pm2 status
pm2 logs bayern-fan-api
pm2 logs bayern-fan-worker
```

重启：

```bash
pm2 restart bayern-fan-api
pm2 restart bayern-fan-worker
```

## 9. 配置 Nginx 反向代理

创建配置：

```bash
sudo nano /etc/nginx/sites-available/bayern-fan-api
```

写入：

```nginx
server {
    listen 80;
    server_name api.example.com;

    client_max_body_size 15m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_connect_timeout 60s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/bayern-fan-api /etc/nginx/sites-enabled/bayern-fan-api
sudo nginx -t
sudo systemctl reload nginx
```

如果有域名，建议再配置 HTTPS：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d api.example.com
```

## 10. 接口联调

健康检查：

```bash
curl https://api.example.com/health
```

提交任务：

```bash
curl -X POST "https://api.example.com/api/v1/synthesis/submit" \
  -H "Content-Type: application/json" \
  -H "x-api-key: 你的_SERVER_API_KEY" \
  -d '{
    "star_ids": ["101", "105", "108"],
    "scene_id": "scene_03",
    "user_image": "data:image/jpeg;base64,这里换成真实图片base64"
  }'
```

成功返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task_xxxxxxxxxxxxxxxx",
    "status": "processing"
  }
}
```

查询任务：

```bash
curl -H "x-api-key: 你的_SERVER_API_KEY" \
  "https://api.example.com/api/v1/synthesis/query/task_xxxxxxxxxxxxxxxx"
```

完成后返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task_xxxxxxxxxxxxxxxx",
    "status": "completed",
    "results": [
      {
        "player_names": ["Alphonso Davies", "Luis Díaz", "Manuel Neuer"],
        "image_url": "https://..."
      }
    ],
    "error": null
  }
}
```

## 11. H5 对接注意事项

H5 后端提交参数：

```json
{
  "star_ids": ["101", "105", "108"],
  "scene_id": "scene_03",
  "user_image": "data:image/jpeg;base64,..."
}
```

H5 后端请求头：

```http
Content-Type: application/json
x-api-key: 生产环境 SERVER_API_KEY
```

当前服务回调 H5 的成功格式已经按交接文档调整为：

```json
{
  "task_id": "task_xxxxxxxxxxxxxxxx",
  "user_image": "https://..."
}
```

建议 H5 回调接口返回：

```json
{
  "code": 0,
  "msg": "success"
}
```

交接文档没有定义失败回调结构，因此当前只发送成功结果回调。失败原因保留在查询接口和 Worker 日志中。

## 13. 常见问题

### 13.1 Redis 连接失败

报错：

```text
connect ECONNREFUSED 127.0.0.1:6379
```

处理：

```bash
sudo systemctl enable --now redis-server
sudo systemctl restart redis-server
redis-cli ping
```

如果使用云 Redis，检查 `.env` 中的 `REDIS_URL`、密码、白名单和安全组。

### 13.2 启动时报缺少环境变量

检查：

```bash
cd /opt/bayern-fan-photo/server
cat .env
```

native 模式至少需要：

- `SEEDREAM_NATIVE_API_KEY`
- `SEEDREAM_NATIVE_API_URL`
- `LAS_API_KEY`
- `LAS_BASE_URL`

其中前两个用于出图，后两个用于用户照片解读。

### 13.3 请求返回 401

说明服务开启了 API Key 校验。H5 后端请求必须带：

```http
x-api-key: 生产环境 SERVER_API_KEY
```

### 13.4 上传 Base64 报 413

Nginx 或 Express 请求体大小限制不够。

Nginx 已建议设置：

```nginx
client_max_body_size 15m;
```

代码中 Express 当前限制是 `10mb`，在 `server/src/app.js`：

```js
app.use(express.json({ limit: '10mb' }));
```

如果用户照片 Base64 超过限制，需要同步调大。

### 13.5 回调失败

检查：

- `H5_CALLBACK_URL` 是否能从服务器访问
- H5 回调接口是否允许服务器 IP
- H5 回调接口是否返回 `{ "code": 0 }`
- PM2 Worker 日志中是否有 `回调失败`

### 13.6 图片生成成功但 H5 展示不了

当前结果图 URL 来自第三方生成服务，可能有有效期或访问限制。正式活动建议补充 OSS/CDN 上传能力，让接口返回自有稳定 URL。
